# RQ2 Table IV Measurement Framework

This directory measures the Table IV data at CVE granularity first, then
summarizes it by protocol version.

The framework intentionally separates static and dynamic columns:

- `Test Case Failures`: static count from the manifest. For F1/F3, this is the
  modeled added or modified value domain size. For F2/F4/F5, this is `1`.
- `Exploit Test Cases #`: dynamic count from the candidate exploit scenario.
- `Exploit Test Case Generation Time`: Maude `real` time for the candidate
  scenario enumeration.
- `Exploitability Success`: dynamic count from the success scenario that reaches
  the closed state.

Only CVEs whose exploit condition is a completed handshake are included in the
main matrix. Mixed-version CVEs are kept as CVE-level rows and summarized under
`mixed` until the paper table assigns them to TLS 1.2 or TLS 1.3.

## Experiment Plan

RQ2 is split into three experiment families:

- `baseline`: only the CVE trigger behavior deviations are enabled. This
  captures the one-step exploit test case already induced by the tester
  deviation and target deviation.
- `bucket`: the CVE trigger behavior deviations are combined with at least one
  extra RQ1 deviation. These jobs are split by version, RFC scenario category,
  and P pattern so they can be run independently.
- `legacy`: the original zero-or-more all-deviation experiment. This is kept for
  compatibility with older snapshots and should not be used as the new Table IV
  total unless explicitly intended.

The bucket matrix is:

- TLS 1.2: `5246` x `P1..P5`
- TLS 1.3: `8446`, `hrr`, `psk` x `P1..P5`

The Table IV CVE-level value is computed as:

```text
baseline + sum(all selected bucket jobs)
```

For the full table, run every bucket for every handshake-success CVE.

## Scenario Shape

Each generated CVE module has legacy properties:

```maude
op rq2Cve...CandidateScenario : -> ScenarioProperty .
op rq2Cve...SuccessScenario : -> ScenarioProperty .
```

They use the old zero-or-more shape:

```maude
(anyStep *) ;
<CVE trigger step 1> ;
(anyStep *) ;
<CVE trigger step 2> ;
(anyStep *) ;
(allDeviationStep *) ;
(anyStep *) ;
<terminal-or-closed condition>
```

New Table IV measurements use these generated properties:

```maude
op rq2Cve...BaselineCandidateScenario : -> ScenarioProperty .
op rq2Cve...BaselineSuccessScenario : -> ScenarioProperty .
op rq2Cve...Tls13PskP3CandidateScenario : -> ScenarioProperty .
op rq2Cve...Tls13PskP3SuccessScenario : -> ScenarioProperty .
```

The baseline shape is:

```maude
(anyStep *) ;
<CVE trigger label sequence> ;
(anyStep *) ;
<terminal-or-closed condition>
```

The bucket shape requires at least one extra RQ1 deviation after the CVE
trigger:

```maude
(anyStep *) ;
<CVE trigger label sequence> ;
(anyStep *) ;
bucketDeviationStep ;
((anyStep *) ; bucketDeviationStep) * ;
(anyStep *) ;
<terminal-or-closed condition>
```

## Manifest Design

`manifest.json` is generated by `scripts/generate_rq2_modules.py`.

Top-level fields:

- `schema_version`: manifest schema id.
- `generated_at`: UTC timestamp.
- `all_deviation_counts`: RQ1 deviation instance totals used by RQ2.
- `buckets`: version/category/P bucket definitions.
- `cves`: CVE rows used by the runner.
- `excluded`: CVEs intentionally not run because they are rejection/crash-only
  or otherwise outside the handshake-success Table IV scope.

Each `buckets[]` row contains:

- `bucket_id`, for example `tls13-psk-p3`
- `protocol`, `aggregation_protocol`, `protocol_version`
- `category`: one of `5246`, `8446`, `hrr`, `psk`
- `pattern`: one of `P1..P5`
- `set_op`, `step_op`
- `expected_instances`
- source `job_ids`, `chunk_ids`, and `source_ids`

Each `cves[]` row contains:

- `case_id`, `cve`, `library`
- `protocol`: `tls12`, `tls13`, or `mixed`
- `aggregation_protocol`: protocol bucket for summaries
- `protocol_version`: Maude protocol argument, for example `TLS-13`
- `final_version`: `v2` or `v3`, used for terminal/closed state predicates
- `module_file`, `module_name`: generated RQ2 CVE module
- `cve_source_file`, `cve_source_module`: original CVE module
- `initial_cwa`, `init_conf`
- `exploit_bds_op`: CVE-specific behavior deviations combined with all RQ1
  deviations
- `rq2_bds_op`, `candidate_property_op`, `success_property_op`
- `baseline_bds_op`, `baseline_candidate_property_op`,
  `baseline_success_property_op`
- `bucket_jobs`: per-CVE bucket measurement jobs
- `prefix_steps`: the ordered trigger step sequence
- `test_case_failures_static`, `failure_classes`, `count_rule`
- `notes`

## Workflow

Generate modules and the manifest:

```sh
python3 maude/scenario/rq2/scripts/generate_rq2_modules.py
```

List runnable CVE/mode jobs:

```sh
python3 maude/scenario/rq2/scripts/run_rq2_matrix.py \
  --list-jobs \
  --experiments baseline bucket \
  --no-generate
```

Run a small syntax/count smoke:

```sh
python3 maude/scenario/rq2/scripts/run_rq2_matrix.py \
  --cves cve-2025-11935 \
  --experiments bucket \
  --categories psk \
  --patterns P5 \
  --modes candidate \
  --cap 1 \
  --timeout 120 \
  --tag bucket-smoke \
  --no-generate
```

Run one split of the Table IV matrix:

```sh
python3 maude/scenario/rq2/scripts/run_rq2_matrix.py \
  --cves all \
  --experiments bucket \
  --categories psk \
  --patterns P1 \
  --modes both \
  --tag table-iv-psk-p1 \
  --timeout 86400 \
  --parallel 1 \
  --no-generate
```

Run the full new Table IV matrix:

```sh
python3 maude/scenario/rq2/scripts/run_rq2_matrix.py \
  --cves all \
  --experiments baseline bucket \
  --categories all \
  --patterns all \
  --modes both \
  --tag table-iv-extra \
  --timeout 86400 \
  --parallel 1 \
  --no-generate
```

Run the old legacy experiment against an existing snapshot:

```sh
python3 maude/scenario/rq2/scripts/run_rq2_matrix.py \
  --cves all \
  --experiments legacy \
  --modes both \
  --tag table-iv-legacy \
  --timeout 86400 \
  --parallel 1 \
  --no-generate
```

Summarize an existing JSONL file:

```sh
python3 maude/scenario/rq2/scripts/summarize_rq2.py \
  --results maude/scenario/rq2/state/jsonl/rq2-matrix.jsonl
```

## Output Layout

- `state/raw/*.maude`: generated Maude driver files.
- `state/raw/*.out`: Maude stdout.
- `state/raw/*.err`: Maude stderr.
- `state/jsonl/*.jsonl`: per-run measurement records.
- `state/csv/rq2-cve-summary.csv`: CVE-level Table IV draft.
- `state/csv/rq2-version-summary.csv`: protocol-level aggregation.
- `state/csv/rq2-bucket-summary.csv`: baseline/bucket component summary.
- `state/md/rq2-summary.md`: Markdown report for inspection.
