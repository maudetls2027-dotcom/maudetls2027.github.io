# TLS-Anvil Bug Expressiveness Table

`results.zip`에서 확인한 TLS-Anvil 실패 항목을 현재 behavior deviation specification 및 scenario property 기준으로 분류한 표이다.

> `wolfSSL server debug-x` 항목은 아카이브 안에 정확한 wolfSSL 버전이 인코딩되어 있지 않았다.

| 분류 | 발생 라이브러리/버전 | 대표 TLS-Anvil 테스트 | 표현성 |
|---|---|---|---|
| malformed handshake/extension length fields | MatrixSSL 4.3.0 client | `certificateMessageLengthTLS12`, `helloExtensionsLengthTLS12`, `helloLengthTLS12`, `serverKeyExchangeLength` | 가능. 현재 `MsgSize`의 symbolic invalid length(`smaller`, `larger`, `maxSize`, `minSize`)로 표현 가능. 정확한 바이트 길이(`-1` 등)는 불가. |
| RFC6066 max_fragment_length negotiation/enforcement | BearSSL 0.6 server; Mbed TLS 2.25.0 server | `respectsNegotiatedMaxFragmentLength` | 현재 불가. `max_fragment_length` extension, record fragment size bound, negotiated extension state를 추가하면 중간 규모 패치로 가능. |
| RFC6066 unrequested CertificateStatus/status_request | BearSSL 0.6 server; Mbed TLS 2.25.0 server | `doesNotSendUnrequestedCertificateStatus` | 현재 불가. `status_request` extension과 `certificate_status` handshake message를 추가하면 중간 규모 패치로 가능. |
| TLS 1.2 close_notify / socket close behavior | GnuTLS 3.7.0 client; s2n 0.10.24 client | `closeNotify` | 현재 불가. TLS alert state와 TCP/socket close observation이 모델 밖에 있음. close state를 추가하면 일부 가능하지만 실제 socket close는 구조적 확장 필요. |
| TLS 1.2 client rejects invalid ServerHello choices/extensions | GnuTLS 3.7.0 client; MatrixSSL 4.3.0 client; s2n 0.10.24 client | `selectUnproposedCompressionMethod`, `sendAdditionalExtension` | 부분 가능. 제안되지 않은 compression/cipher/extension 선택은 메시지 필드 및 extension set 변조로 가능. 미지원 extension의 정확한 종류는 현재 extension vocabulary에 의존. |
| TLS 1.2 client ServerKeyExchange validation | s2n 0.10.24 client | `acceptsAnonSignatureForNonAnonymousCipherSuite`, `acceptsMissingSignature`, `acceptsUnproposedNamedGroup`, `acceptsUnproposedSignatureAndHash`, `invalidServerKeyExchangeSignature` | 대체로 가능. signature/hash/group 변조와 `noCheck` 계열로 표현 가능. 단, missing signature 자체는 작은 syntax/semantics 패치가 있으면 더 직접적으로 표현 가능. |
| client Certificate message required after CertificateRequest | MatrixSSL 4.3.0 client | `clientMustSendCertMsg` | 가능. `skip(#certificate)` 또는 state/sequence property로 CertificateRequest 이후 Certificate 생략을 표현 가능. |
| undefined TLS record content types | MatrixSSL 4.3.0 client; MatrixSSL 4.3.0 server; s2n 0.10.24 client; wolfSSL 4.5.0 server | `sendNotDefinedRecordTypesWithCCSAndFinished`, `sendNotDefinedRecordTypesWithServerHello` | 부분 가능. 현재 content type sort가 유한해서 진짜 unknown type은 불가. `ContentType`에 unknown/invalid 값을 추가하면 작은 패치로 가능. |
| record crypto / CCS / Finished workflow | MatrixSSL 4.3.0 client; MatrixSSL 4.3.0 server | `ccsContentTest`, `checkReceivedMac`, `happyFlow`, `invalidAuthTag`, `verifyFinishedMessageCorrect` | 혼합. Finished 검증 누락/순서 오류는 가능. AEAD tag, CBC MAC, CCS payload byte 수준 검사는 현재 추상화 밖이라 불가. |
| empty/zero-length records and fragmentation | MatrixSSL 4.3.0 client; MatrixSSL 4.3.0 server | `sendEmptyApplicationRecord`, `sendEmptyFinishedRecord`, `sendZeroLengthApplicationRecord` | 현재 불가. record fragmentation, empty fragment, encrypted inner plaintext length가 모델 밖에 있음. record-fragment layer를 추가해야 함. |
| TLS 1.2 version compatibility | MatrixSSL 4.3.0 server; wolfSSL 4.5.0 server | `acceptAnyRecordVersionNumber`, `versionGreaterThanSupportedByServer` | 부분 가능. TLS-1.0~1.3 범위의 version mismatch는 가능. SSL3 또는 임의 record version은 `ProtocolVersion` 확장이 필요. |
| TLS 1.2 resumption/SNI/fatal-alert session state | LibreSSL 3.2.3 server; MatrixSSL 4.3.0 server; wolfSSL 4.5.0 server; wolfSSL server debug-x | `rejectResumptionAfterFatalPostHandshake`, `rejectSniDisparityResumption`, `serverHelloSniInResumption` | 부분 가능. resumption/fatal-alert 상태 전이는 일부 가능. SNI는 현재 extension vocabulary에 없어서 작은-중간 패치 필요. |
| TLS 1.2 RSA premaster malformed input | MatrixSSL 4.3.0 server | `PMWithWrongClientVersion`, `PMWithWrongPKCS1Padding` | 부분 가능. premaster version mismatch는 가능. PKCS#1 padding 바이트 구조는 crypto/parser 추상화 밖이라 현재 불가. |
| post-handshake second ChangeCipherSpec state-machine | LibreSSL 3.2.3 server | `secondChangeCipherSpecAfterHandshake` | 현재는 제한적. CCS 메시지 자체는 있으나 post-handshake second CCS acceptance를 직접 관찰하려면 handshake-complete state와 CCS acceptance/error property 보강이 필요. 작은-중간 패치로 가능. |
| TLS 1.2 server ClientHello variation handling | MatrixSSL 4.3.0 server; wolfSSL 4.5.0 server; wolfSSL server debug-x | `includeUnknownExtension`, `leaveOutExtensions`, `offerManyCipherSuites`, `unknownCipherSuite`, `unknownCompressionMethod` | 부분 가능. extension 추가/삭제, cipher suite list 변형은 가능. unknown cipher/compression은 vocabulary 확장이 필요. |
| TLS 1.2 server ServerHello structure/random checks | MatrixSSL 4.3.0 server; wolfSSL 4.5.0 server | `checkExtensions`, `serverRandom` | 부분 가능. extension 구조/선택 오류는 가능. server random의 concrete entropy/byte pattern 검사는 현재 불가. |
| TLS 1.2 server signature/SKE handling | LibreSSL 3.2.3 server; MatrixSSL 4.3.0 server; wolfSSL 4.5.0 server; wolfSSL server debug-x | `includeUnknownSignatureAndHashAlgorithm`, `offerManyAlgorithms`, `signatureIsValid` | 가능/부분 가능. signature algorithm list 및 signature validation deviation은 가능. unknown signature/hash 값은 vocabulary 확장이 필요. |
| RFC8422 point-format/chosen-curve behavior | MatrixSSL 4.3.0 server; wolfSSL server debug-x | `deprecatedFormat`, `invalidPointFormat`, `respectChosenCurve`, `respectChosenCurveWithoutFormats`, `serverAdvertisesOnlyUncompressedPointFormat` | 부분 가능. chosen group/curve mismatch는 일부 가능. `ec_point_formats` extension과 point-format values가 없어서 point-format 계열은 작은-중간 패치 필요. |
| RFC8422 supported-group/invalid-curve behavior | MatrixSSL 4.3.0 server; wolfSSL server debug-x | `addUnknownEllipticCurve`, `invalidEllipticCurve_WithNonECCCiphersuite`, `manyGroupsOffered`, `rejectsInvalidCurvePoints`, `supportsDeprecated` | 부분 가능. supported groups list 조작은 가능. invalid curve point validation은 elliptic-curve point arithmetic/encoding이 모델 밖이라 현재 불가. |
| TLS 1.3 rustls RFC8446 failures | rustls 0.19.0 server | `invalidLegacyVersion_ssl30`, `sendPskExtensionWithoutPskKeyExchangeModes`, `signatureIsValid`, `zeroLengthRecord_Finished` | 혼합. PSK/key_exchange_modes와 signature validation은 가능. SSL3 legacy version은 vocabulary 확장 필요. zero-length encrypted Finished record는 record-fragment layer가 없어 불가. |
